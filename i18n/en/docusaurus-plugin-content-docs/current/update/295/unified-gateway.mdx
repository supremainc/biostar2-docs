This document provides an overview of the BioStar 2 Unified Gateway and offers guidance on its usage.

## Unified Gateway

BioStar 2 has a structure where AC and TA web pages with different structures access one server, and by setting up a <Cmd sid="common.enabled" />, you can change the execution port of BioStar 2 to one Unified Gateway.

<Cmd sid="common.enabled" /> allows efficient processing of requests to the BioStar 2 server through a reverse proxy method, improves security vulnerabilities in iframes, and minimizes SSL certificate errors.

:::info

* <Cmd sid="common.enabled" /> is available from BioStar 2 v2.9.3.

* When installing or upgrading BioStar 2, the <Cmd sid="common.enabled" /> is <Cmd>Inactive</Cmd>, and the default port values are as follows:

    * <Cmd sid="port.gatewayHttps" />: 5002

    * <Cmd sid="port.gatewayHttp" />: 5000

* When applying an SSL certificate in an environment using <Cmd sid="common.enabled" />, restart <Cmd sid="common.enabled" />.

* <Cmd sid="common.enabled" /> is not available in environments using <Cmd sid="cloud.title" />.

:::

### Setting up a Unified Gateway in the BioStar Setting

1. Run <Start/> → <Cmd sid="common.biostar" /> → **BioStar Setting**.

1. Click the **Start** button in the **Unified Gateway** section. <Cmd sid="common.enabled" /> becomes active.

    :::info

    * Activating <Cmd sid="common.enabled" /> will restart the server.

    * If the port is in use, a pop-up message will appear. Enter a different port number and activate it again.

    :::

    <Image src='/img/common/biostar-setting.png' className='none' caption />

1. If the HTTPS connection, the **HTTPS Port** field changes to 5002, and the **Unified Gateway**’s **Port** field changes to 443.<br/>To change the port, click **Stop** to stop **Unified Gateway**, and then change it.

    <Image src='/img/common/biostar-setting-unified-gateway.png' caption />

### Setting up a Unified Gateway in BioStar 2

1. Access to <Cmd sid="common.biostar" />\.

1. Click <Cmd sid="common.setting" /> → <Cmd sid="setting.systemSecurity" />.

1. Activate the <Cmd sid="common.enabled" /> in the <Cmd sid="setting.systemSecurity.sessionGateway" />.

    <Image src='/img/biostar-setting-security-unified-gateway.png' className='none' caption />

:::info

* When you activate <Cmd sid="common.enabled" />, the server restarts and is automatically directed to the login page.

* If the port is in use, a pop-up message will appear. Enter a different port number and activate it again.

:::
