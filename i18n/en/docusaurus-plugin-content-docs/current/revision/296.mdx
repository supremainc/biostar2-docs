## New Features and Improvements

---

1. Supports a new device.

    * XPass 2 (XP2-MAPB)

1. Supports the **Display Result from Controller** feature for displaying authentication success or failure results from a 3rd party controller on the device screen when using **Intelligent Slave**.

1. Supports the **Ignore Repeated Signals Duration** feature, which ignores repetitive authentication signals from the controller when the device is connected to a 3rd party controller via Wiegand.

1. Supports the feature to select the resolution of the video transmitted from the device when using **Intercom** or **RTSP**.

    * **Intercom Video Resolution**

    * **RTSP Video Resolution**

1. The authentication result display option supports selecting how the user ID and name are displayed on the device’s authentication result screen.

    * **User ID Display**

    * **User Name Display**

1. Supports **SIP Server Transport**, which selects the SIP transmission method when setting up the SIP server of the **Intercom** on the device.

1. Supports **Auth Mode** editing in **Batch Edit** of devices.

1. FaceStation F2 (firmware v2.1.4 or higher) - Added **Power Line Frequency** option to prevent flicker based on the illuminance environment in which the device is installed.

1. Supports FeliCa card from **Custom Smart Card Layout**.

1. The **Face Detection** feature on X-Station 2 is discontinued.

1. Improved security vulnerabilities.

    * Improved XSS security vulnerability.

    * Improved web socket security vulnerabilities.

    * Improved CSRF security vulnerability.

    * Upgraded 7zip version.

    * Improved security vulnerabilities of SSL/TLS Diffie-Hellman Modulus.

    * Upgraded Spring Framework version.

    * Improved security vulnerabilities of HTTP Content-Security-Policy.

    * Removed the auto-completion feature from all input fields.

    * Improved SQL Injection security vulnerabilities.

    * Improved BioStar 2 certificate security vulnerabilities.

    * Improved **Active Directory** security vulnerabilities.

1. Improved Time and Attendance security vulnerabilities.

    * Improved security vulnerabilities of Access-Control-Allow-Origin.

    * Added X-Frame-Options to T&A API document.

1. Improved user inquiry screen performance in the **TIME ATTENDANCE** → **Schedule** menu.

1. Added a refresh button to the T&A record details view screen (calendar, list) in the **TIME ATTENDANCE** → **Report** menu.

1. Improved time display method in the T&A Report in countries that do not use Daylight Saving Time (DST).

1. Improved synchronization performance of the latest data when there is a large volume of old T&A report data.

1. Improved to display both the user name and user ID in the **User** column of the **Alert List**.

1. Improved the time to be displayed in seconds in the **Date** column of the **Alert List**.

1. Improved performance when Automatic User Synchronization is set to Specific Devices(Only devices belonging to the access group) option.

    * Improved performance when adding, modifying, or deleting **ZONE**.

    * Improved **Access Level** edit performance.

1. Improved **User Group** and **Device Group** synchronization of **TIME ATTENDANCE**.

1. Fixed error pop-up message is displayed when uploading visual face files with filenames starting with ‘0’ while the **User ID Type** is set to **Number**.

1. Improved check logic for duplicate user IDs when using **User ID Type** as **Alphanumeric**.

1. Improved to use a static IP address when calling internal APIs from C to Java.

1. Increased the maximum number of authentication modes for a specific device.

1. Added new index for environments using MS SQL Server databases.

1. Adjusted the maximum number of characters that can be entered in the card ID when issuing a CSN mobile card.

1. Improved to display a pop-up message when the user sets the encryption key storage path directly with special characters in the path.

1. Improved Wiegand reader to be supported in **TRIGGER & ACTION**.

1. Improved to check the server’s readiness before starting the BioStar 2 server and display a popup message in case of any issues.

1. Improved the display of user-related event logs in **CSV Export** or **Print** to differentiate whether the event occurred from the device or server.

1. Improved to leave an event log and alert in real-time when **Maximum Invalid Attempts** is exceeded.

1. Added API to edit information in batch up to 100 users.

1. Improved password **Maximum Invalid Attempts** limit option to be applied by default value.

1. Added notification pop-up messages for unset items and reduced the conditions required when creating an **Access Group**.

1. Improved login API.

1. Improved mobile card ID check logic.

1. Improved BioStar 2 Setting UI

1. Improved display in the status window of BioStar 2 Setting when the service ports are in use.

1. Updated Spanish resource file.
