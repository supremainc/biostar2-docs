Release<Separator/>2020-12-29

## New Features and Improvements

---

1. Stabilization of features for Mobile Access.

    * Supports Dynamic site.

    * Displays user information, such as photo, department, title, and period on the Mobile Access Cards.

    * Improves a user interface for resending Mobile Access Cards.

1. Provides a guide on the screen when registering Visual Face on the mobile device.

1. Renamed 'Email Setting' menu.

    * Before: Email Contents

    * After: Email Setting

1. Renamed some events related to scheduled unlock zones.

1. Updated some language resource files.

1. Changed the names of db-converter log files.

1. Changed to possible to edit the device's T&A settings only while the device is connected.

## Main Fixes

---

<BugLists>

1. Extended auth mode gets disabled intermittently when multiple FaceStation F2 devices try to connect to the server simultaneously. <Badge>Affects version: v2.8.8</Badge>

1. Punch logs were not displayed if the punch type was null in the updated individual reports by selecting the 'All Punches' option. <Badge>Affects version: v2.7.10</Badge>

1. Punch logs were not recorded properly in the individual reports updated by selecting the 'All Punches' option. <Badge>Affects version: v2.7.10</Badge>

    * The 'Check In' log was only recorded in the report even though the user selected 'Check Out' on the T&A device.

1. BioStar 2 could not connect with the TA database if using a port other than the default port when installed using MS SQL Windows authentication. <Badge>Affects version: v2.7.14</Badge>

1. PIN authentication failed due to the device default hash key transmission error. <Badge>Affects version: v2.4.0</Badge>

1. If both user data registered with a face template and user data registered with a visual face template were sent to FaceStation 2 and FaceStation F2 at the same time, the authentication failed on FaceStation F2. <Badge>Affects version: v2.8.6</Badge>

1. T&A device synchronization problems

    * Devices added in DEVICE menu were not synchronized with the TIME ATTENDANCE menu.

    * FaceLite added in DEVICE menu were not displayed in the T&A device list. <Badge>Affects version: v2.7.10</Badge>

    * Devices added in DEVICE menu were not displayed in the T&A device list because the value of device group ID became null due to a synchronization problem. <Badge>Affects version: v2.7.5</Badge>

1. When starting the T&A service, a user was automatically signed out of BioStar 2 and was not able to sign in again. <Badge>Affects version: v2.8.4</Badge>

    * When accessing the T&A service while a user was signed in to BioStar 2, 'Unknown error. Check the log file for detailed information' error message was displayed.

1. If clicking Apply in the SERVER menu after setting the mobile access card option to 'Use', the setting was changed to 'Not Use'. 

1. An email notification about users who have reached their specified working hours was not sent even after configuring the SMTP setting of Automated Email. (Affects version: v2.8.6, v2.8.8)

</BugLists>

## Bug Fixes

---

<BugLists>

1. An error occurred when creating work history after running user sync with personal information DB turned on or off. <Badge>Affects version: v.2.8.0</Badge>

1. When issuing mobile cards, model names of some devices were not correctly displayed in the pop-up message. <Badge>Affects version: v2.6.0</Badge>

1. Delay occurred when uploading event logs related with temperature from FaceStation F2 to the server and unnecessary logs, such as temperature data with zero value, were uploaded. <Badge>Affects version: v2.8.6</Badge>
